Reading guide — Microsoft poster visual conventions

Containers

Light blue dashed rounded rectangles indicate logical groupings (service boundaries, tenant scopes). Always labeled in the upper-left corner.

Component boxes

Solid blue fills (dark / mid / light) for Microsoft services. White boxes with thin blue borders for sub-components. Pink/magenta connectors with terminal dots indicate data relationships.

Trust boundaries

Red dashed lines (Shostack DFD3 standard) indicate trust-boundary crossings — by definition every crossing is an attack surface. Added in §3 for threat-modeling use.

Microsoft Copilot for Microsoft 365
service and tenant logical architecture

Your customer data stays within the Microsoft 365 service boundary. Your prompts, responses, and data in the Microsoft Graph are not used to train foundation LLMs.

Your customer data stays within the Microsoft 365 service boundary. Your prompts, responses, and data in the Microsoft Graph is not used to train foundation LLMs that Copilot leverages. Your data is secured based on existing security, compliance, and privacy policies already deployed by your organization.

Your tenant sits inside the Microsoft 365 service boundary, where Microsoft's commitment to security, compliance, data location, and privacy are upheld.

Copilot is a shared service just like many other services in Microsoft 365. Communication between your tenant and Copilot components is encrypted.

10-stage Copilot interaction
STRIDE risk assessment

Every Copilot interaction passes through the same ten stages. Each is a chance to add value — and a place where something can go wrong. Adapted from John Farag's STRIDE executive brief (May 2026), expanded with OWASP LLM Top 10 (2025) mappings.

1 User signs in

Entra ID authenticates the user, evaluates Conditional Access, issues an OAuth 2.0 access token. Token lifetime: 60–90 minutes default (configurable); refresh tokens default to 90 days, revoked on policy change, password reset, or risk detection.

SE STRIDE threats

Spoofing: Stolen credentials, AiTM, cookie theft.

Elevation of Privilege: Tricked MFA gives attacker the user's full Copilot scope.

E5 controls

  • Entra ID P2 — risk-based Conditional Access
  • Identity Protection — auto-detect compromise
  • PIM — JIT admin elevation
  • FIDO2 — defeats AiTM

Business impact

Copilot is only as trustworthy as the identity using it. E3 gives basic MFA; E5 gives the risk-based brain that decides when MFA is enough and when to block.

OWASP: LLM02

2 User types a prompt

Prompt and active context packaged on the user's device. Encrypted in transit over TLS 1.2/1.3 via Microsoft Global Network.

TI STRIDE threats

Tampering: Endpoint malware injects extra instructions before egress.

Info Disclosure: Keylogger captures sensitive prompt content.

E5 controls

  • Defender for Endpoint P2 — full EDR
  • Endpoint DLP — blocks regulated data in prompts
  • Defender XDR — cross-domain correlation

Business impact

The keyboard is the new front door for AI. If the device is dirty, every prompt and answer is at risk — no matter how secure the cloud is.

OWASP: LLM02

3 Copilot prepares the prompt

Orchestrator reads prompt + local context. Azure AI Content Safety + Prompt Shields scan input.

TI STRIDE threats

Tampering (indirect prompt injection): Hidden instructions in document, email, web page hijack Copilot.

Info Disclosure: Copilot pulls in highly sensitive content without user awareness of scope.

E5 controls

  • Defender for O365 P2 — Safe Links / Attachments
  • Purview auto-labeling
  • Insider Risk Mgmt — unusual context patterns
  • Prompt Shields — built-in

Business impact

Prompt injection is the new phishing — OWASP's #1 LLM risk for 2025.

OWASP: LLM01 (primary), LLM02

4 Copilot looks up your data (Graph grounding)

Hybrid keyword + vector retrieval against Graph index, executing as the user. ACLs enforced at the index layer.

IE STRIDE threats

Info Disclosure (oversharing — #1 Copilot risk): Loose SharePoint permissions exposed. The classic HR salary file with open perms — suddenly summarized.

EoP (data plane): Compromised account = tenant-wide search engine.

E5 controls

  • SharePoint Advanced Mgmt — RSS, RAC, RCD, DAG
  • Purview DLP (full coverage)
  • Auto-labeling at scale
  • Insider Risk Mgmt

Business impact

Copilot doesn't break permissions — it exposes them. Cleaning up oversharing after a rollout costs far more than fixing it before launch.

OWASP: LLM02 (primary), LLM08

5 Copilot builds the full request

System prompt + user prompt + grounded data assembled into one request. Sent to Azure OpenAI inside the M365 service boundary.

TI STRIDE threats

Tampering: Sensitivity labels stripped — encryption lost in transit.

Info Disclosure: Confidential content mixed with low-trust context (public web, guest chat).

E5 controls

  • Purview Info Protection — label inheritance
  • Purview DLP — halt the request entirely if needed
  • Customer Key / DKE (E5 add-ons)

Business impact

Labels and DLP turn Copilot from a free-for-all assistant into a compliant one. Without E5 auto-labeling at scale you cannot enforce these consistently.

OWASP: LLM07, LLM05

6 The AI model generates an answer

Azure OpenAI processes the assembled prompt inside Microsoft's commercial data boundary. Tenant data not retained.

ID STRIDE threats

Info Disclosure (output exfil): Prompt injection causes model to embed sensitive data in a markdown image link.

Denial of Service: Prompt floods, expensive queries (LLM10) degrade performance.

E5 controls

  • Defender for Cloud Apps — shadow GenAI discovery
  • Defender XDR — abuse signal correlation
  • MS Copilot service controls — rate limit

Business impact

The model is hardened by Microsoft, but how users access it (devices, browsers, copy-paste) is your responsibility.

OWASP: LLM10, LLM05

7 The answer is checked (Responsible AI)

Before reaching the user, Copilot runs Responsible AI checks — harmful content filters, citation generation, label inheritance.

TI STRIDE threats

Tampering: Jailbreak bypasses safety filters or strips citations.

Info Disclosure: Unfiltered output includes confidential data without label or banner.

E5 controls

  • Copilot Responsible AI — built-in
  • Purview Comm Compliance (E5-only) — flags AI-leaked content, regulated phrases, IP leakage
  • Purview Info Protection — label inheritance

Business impact

RAI isn't only about offensive content. E5 Comm Compliance catches the quiet risks: confidential data leaking into a customer email.

OWASP: LLM09, LLM01

8 The answer is shown to the user

Copilot displays the answer. User can accept, edit, copy, or discard.

IR STRIDE threats

Info Disclosure: User copies the answer into an unmanaged channel.

Repudiation: No one can prove what Copilot showed, what grounded it, or whether the user acted on it.

E5 controls

  • Endpoint DLP — block copy/paste/screenshot
  • Defender for Cloud Apps — browser session controls
  • Purview Audit (Premium) — full prompt/response/grounding; 1-year retention

Business impact

The "screen" is where data leaves the secure boundary. Without E5 audit and DLP, you cannot prove what Copilot told whom.

OWASP: LLM02, LLM05

9 Copilot takes action on your behalf

Beyond answering, Copilot sends emails, posts in Teams, updates docs, schedules meetings, modifies SharePoint files.

ETR STRIDE threats

EoP: Compromised account scales insider attack at machine speed.

Tampering: Automated edits to important documents without review.

Repudiation: No proof of who really triggered the action.

E5 controls

  • Defender for O365 P2 — AIR for mass-mail
  • Defender for Cloud Apps — mass-modification anomaly
  • Insider Risk Mgmt — abnormal action sequences
  • PIM — JIT for admin scopes

Business impact

Copilot is the first AI in your tenant that can act, not just answer. Blast radius of a compromised account doubles. E5 shrinks it back down.

OWASP: LLM06 (primary), LLM01

10 Everything is logged

Every Copilot interaction recorded in the unified audit log. DSPM for AI exposes prompt/response content for E5 admins.

RT STRIDE threats

Repudiation: Without rich long-lived logs, no proof of Copilot activity.

Tampering: Logs "aged out" before investigation finishes.

E5 controls

  • Purview Audit (Premium) — 1-year retention; Copilot Interactions event is E5-only
  • Purview eDiscovery (Premium)
  • Data Lifecycle / Records Mgmt
  • Microsoft Sentinel — long-term SIEM correlation

Business impact

If you cannot answer "who asked Copilot what, when, and from which data source?" a year later, you have a compliance problem.

OWASP LLM Top 10 (2025)
overlay onto Copilot

STRIDE was designed in 2002 for classic application threats. LLM applications introduce risks that don't fit cleanly — which is why OWASP publishes the LLM Top 10. Each risk is mapped onto the Copilot stage where it manifests and the E5 control that mitigates it.

LLM
01
Prompt Injection — #1 risk, both 2023 & 2025

Direct or indirect manipulation of model behavior. LLMs process instructions and data in the same channel — they can't tell the difference. Surfaces at Copilot stages 3, 5, and indirectly 6.

E5 mitigation: Defender for O365 P2 (Safe Links / Attachments), Prompt Shields, auto-labeling, App Governance for plugins.
LLM
02
Sensitive Information Disclosure — up from #6 to #2

LLM unintentionally reveals private/proprietary information through outputs, memorization, or retrieval. The Copilot oversharing problem lives here.

E5 mitigation: Purview DLP (full coverage), auto-labeling, SAM, DSPM for AI prompt/response visibility.
LLM
03
Supply Chain — broadened, up from #5 to #3

Vulnerabilities in third-party models, datasets, plugins, MCP servers, agents. Surfaces at Copilot's extensibility surface.

E5 mitigation: Defender for Cloud Apps + App Governance for plugin vetting; admin consent policies.
LLM
04
Data and Model Poisoning

Malicious data injected into training, fine-tuning, or RAG corpus. Lower risk for Copilot but RAG poisoning of tenant Graph index possible via connected SaaS.

E5 mitigation: Defender for Cloud Apps detects unusual connector inflows; IRM; per-connector ACL.
LLM
05
Improper Output Handling — down from #2 to #5

LLM responses passed unsanitized to downstream systems. For Copilot: malicious markdown image link that exfils data when rendered.

E5 mitigation: Azure AI Content Safety output filter, Endpoint DLP on render, Defender for Cloud Apps session controls.
LLM
06
Excessive Agency — significantly expanded in 2025

Excessive functionality, permissions, or autonomy. Copilot stage 9 — "takes action on your behalf" — is this category by construction.

E5 mitigation: PIM for admin scopes, Defender for O365 AIR, Defender for Cloud Apps anomaly, IRM for action sequences.
LLM
07
System Prompt Leakage — new in 2025

System prompt revealed via injection. Less critical for Copilot (system prompt is Microsoft-managed) but custom Studio agents carry this risk.

E5 mitigation: Don't put secrets in system prompts; App Governance for agents; Communication Compliance flags suspicious prompts.
LLM
08
Vector and Embedding Weaknesses — new in 2025

RAG-specific: vector DB poisoning, tenant-boundary leakage in embeddings, embedding model manipulation. Copilot's Semantic Index is the relevant surface.

E5 mitigation: Semantic Index is per-user and tenant-resident by design; ACL at retrieval; SAM tightens indexed content.
LLM
09
Misinformation — renamed from "Overreliance" in 2025

Model hallucinates; application propagates falsehood. Copilot mitigates with citations and groundedness checks, but humans still over-trust.

E5 mitigation: Communication Compliance flags risky AI-generated content; audit captures grounding sources for verification.
LLM
10
Unbounded Consumption — folded from "Model DoS"

Prompt floods, resource-heavy operations cause DoS, model theft, financial loss. Microsoft service-side throttling handles most of this.

E5 mitigation: Microsoft Copilot service controls (rate limits); Defender XDR detects abuse; cost-monitoring via admin reports.
Why this overlay matters. A STRIDE-only analysis misses LLM-native categories — Excessive Agency, Vector Weaknesses, System Prompt Leakage, Misinformation. A pure OWASP LLM analysis misses classic identity threats — Spoofing, Repudiation. Doing both is the current industry best-practice for AI threat modeling, per the OWASP GenAI Security Project.

E3 vs E5 & deployment sequence
Prepare your environment for Microsoft Copilot

Capabilities that materially change Copilot's risk profile, plus the recommended deployment sequence (mirrors Microsoft's official "getting started with E3" / "next steps with E5" framework).

What E5 adds beyond E3 for Copilot

CapabilityE3E5Why it matters for Copilot
Risk-based Conditional Access (Entra ID P2)NoYesBlocks impossible-travel / leaked-credentials sign-ins before Copilot is reached
Privileged Identity Management (PIM)NoYesRemoves standing admin access for Copilot configurators
Defender for O365 P2 (Safe Links/Attachments, AIR)NoYesStops indirect prompt injection at the email/file ingress
Defender for Endpoint P2 (full EDR)NoYesCatches keyloggers, malicious extensions, in-memory attacks targeting prompts
Defender for IdentityNoYesLateral-movement detection in hybrid identity
Defender for Cloud AppsNoYesApp Governance for Copilot plugins; shadow GenAI discovery
Endpoint, Teams & Browser DLPNoYesFull prompt/response coverage; E3 only covers Exch/SP/OD
Information Protection auto-labeling at scaleNoYesEnforces protection on legacy/un-labeled docs Copilot might surface
Insider Risk ManagementNoYesDetects departing-employee abuse patterns and abnormal Copilot search
Communication ComplianceNoYesFlags risky Copilot-generated content
Audit (Premium) — Copilot Interactions, 1-year retention90 days1+ yearOnly E5 has the full Copilot prompt/response audit event
eDiscovery (Premium)NoYesLegal hold and purge of Copilot interactions
DSPM for AI (content-level)activity onlyfull contentE5 admins see what was prompted/answered

Recommended deployment sequence

Microsoft recommends deploying Copilot as you phase in protections for access, devices, and data: (1) apply identity and access protection · (2) apply device protection · (3) assign Copilot licenses to users with these protections.

Key takeaway. E5 is the recommended security baseline for Copilot. Every E5-only control in this document supports at least one Copilot-specific threat that has no equivalent mitigation in E3.
